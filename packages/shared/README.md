# packages/shared

## 用途/职责
跨应用共享的类型、常量与工具集合，降低重复实现；不承载领域规则。

## 包含内容
- 当前为空目录，预留共享类型与跨包工具。

## 与其它目录的依赖关系
- 被依赖方：`apps/web`、`apps/api`、`packages/domain`、`packages/ai-orchestrator`。
- 允许依赖：尽量保持零依赖，如需依赖必须说明原因。
- 禁止依赖：`/apps/*`、`/infra/*`。

## 禁止事项
- 在 shared 中实现业务规则或流程逻辑。
- 写入特定运行环境或第三方平台的耦合代码。

## 常见变更场景
- 新增共享类型：定义清晰边界，避免引入领域语义。
- 迁移工具函数：从应用层抽取通用逻辑并补充说明。

## 命名规范/约定
- 类型命名使用 PascalCase，常量使用全大写下划线。
- 工具函数名保持动词语义且可复用。

## 测试/验证原则
- 验证共享类型不泄漏业务规则。
- 对外暴露接口稳定可复用。
