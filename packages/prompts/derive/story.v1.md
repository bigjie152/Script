## 【板块：故事 Story（复盘叙事层）】
你将获得已锁定的 Truth。你必须 100% 服从 Truth，不得新增事实。

### 目标（商业可用）
- 生成一份可直接用于“主持人复盘/发行审稿”的 Story
- 叙事要清晰、可读、有节奏，但不能用角色第一人称
- 必须把 Truth 的关键事实全部“可回收地呈现”

### 硬性长度/结构要求（不达标视为失败）
- 故事背景 400–700 字
- 事件展开 1200–2000 字（分段，含关键转折点）
- 真相揭示 600–1000 字（按证据链回收，不是抒情总结）
- 必须显式列出 8 个“复盘回收点”（每点对应 Truth 的一个关键事实）

### 输出要求
- 只输出 JSON，不要解释
- content 字段为纯文本（非 Markdown），使用【】标题与分行结构

### 强制输出结构

【Story（复盘叙事）】

【1 故事背景（Public）】
（400–700字：交代世界状态、人物群像的共同处境、当下危机）

【2 事件展开（按叙事顺序）】
（1200–2000字：分 6–10 个短段落。每段结尾给出“叙事钩子”或“疑点”。）
- 段落 1：
- 段落 2：
（…）

【3 真相揭示（复盘文本）】
（600–1000字：以“证据 → 事实 → 推论”的顺序回收，严格服从 Truth）
- 回收 1：
- 回收 2：
（…）

【4 复盘回收点清单（必须 8 条）】
- 回收点 1：对应 Truth 的【事实/时间节点】是什么
- …

【禁止事项】
- 禁止第一人称
- 禁止“玩家调查发现”措辞
- 禁止新增 Truth 未出现的新事实

### Output JSON
{
  "preview": "占位预览",
  "items": [
    {
      "type": "Story",
      "title": "文学复盘总纲",
      "content": "<纯文本内容，必须严格遵守上述结构>"
    }
  ]
}
