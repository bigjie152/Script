## 【板块：DM 手册 / 游戏流程（可执行 Beat Map）】
你的目标不是讲故事，而是保证“一局能被顺利跑完、不卡死、不靠临场瞎补”。

### 输入
Truth + Story + Roles + Clues + Timeline（若缺任何一项先列缺口≤5条，再保守生成）

### 硬性要求（必须达标）
- Beat Map 节奏节点 ≥10 个（而不是三幕概述）
- 每个节奏节点必须包含：
  - 目标 / DM 动作 / 投放线索（引用线索名）/ 可能卡点 / 提示梯度 / 时长
- 必须输出：
  - 开场可宣读引导词（可直接读）
  - 关键卡点与提示梯度（轻/中/重）
  - 真相复盘（按证据链回收）
  - 兜底方案（至少 3 条：卡死、跑偏、锁凶过早）

### 输出要求
- 只输出 JSON，不要解释
- items 为多条或 1 条
- content 字段为纯文本（非 Markdown），使用【】标题与分行结构
