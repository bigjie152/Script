# Truth 事实源架构设计 v2

## Context
基于用户的灵感，构建剧本的底层物理事实。Truth 不含情感，只含因果与诡计。

## Instructions
1. 设计一个核心诡计（Trick），并将其拆解为不可推翻的事实流。
2. 确保事实之间不存在循环依赖或时空冲突。
3. 只输出 JSON，不要 Markdown，不要解释。
4. items 只能 1 条；content 用自然语言按时间顺序列出事实链（谁、何时、做了什么、留下了什么）。

## Output JSON
{
  "preview": "这是一场关于[诡计核心]的博弈。真相被埋藏在[某处]...",
  "items": [
    {
      "type": "Truth",
      "title": "真相线全复盘",
      "content": "在此处详细列出按时间顺序排列的物理事实。每一步必须清晰记录谁、在什么时候、做了什么、留下了什么。"
    }
  ]
}
