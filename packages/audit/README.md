# packages/audit

## 用途/职责
审计与记录能力边界，用于捕获关键操作与领域事件，支持追溯与合规要求。

## 包含内容
- 当前为空目录，预留审计模型、记录策略与接口定义。

## 与其它目录的依赖关系
- 被依赖方：`apps/api`、`packages/database`。
- 允许依赖：`/packages/domain`（事件或实体引用）与 `/packages/shared`。
- 禁止依赖：`/apps/web`、`/infra/*`。

## 禁止事项
- 在 audit 包中实现业务流程或权限规则。
- 将外部存储/平台细节硬编码在审计模型中。

## 常见变更场景
- 新增审计事件：定义事件结构并说明触发条件。
- 调整记录粒度：更新事件字段并同步 API 与数据库映射说明。
- 扩展审计范围：在 Domain 事件中补充可追溯信息。

## 命名规范/约定
- 审计事件命名体现动词与对象（如 `ProjectCreatedEvent`）。
- 字段命名保持稳定、可追踪，避免含糊缩写。

## 测试/验证原则
- 验证关键路径产生完整审计记录。
- 变更需确保可追溯性与合规性不下降。
