# infra/aliyun

## 用途/职责
阿里云基础设施占位目录，用于规划容器、计算与数据库资源，不承载业务规则。

## 包含内容
- docker/：容器镜像与运行时占位
- ecs/：计算资源（ECS）占位
- rds/：数据库资源（RDS）占位

## 与其它目录的依赖关系
- 允许依赖：无代码依赖，仅对接应用部署产物与环境变量需求。
- 禁止依赖：`/packages/*` 与 `apps` 源码实现。

## 禁止事项
- 将业务逻辑、Domain 规则或 Prompt 放入 infra。
- 在 infra 中直接维护业务数据库结构（应在 database 包）。

## 常见变更场景
- 增加资源类型：在 `aliyun/` 下新增目录并补充 README。
- 调整部署参数：记录变更影响与回滚方案。
- 迁移环境：保持与应用契约一致，避免隐式配置。

## 命名规范/约定
- 子目录名称与云服务名称一致（docker/ecs/rds）。
- 配置文件以用途命名，避免模糊缩写。

## 测试/验证原则
- 变更先在测试环境验证资源与访问策略。
- 确保基础设施调整不引入业务逻辑耦合。
