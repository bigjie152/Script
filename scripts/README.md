# scripts

## 用途/职责
运维与迁移辅助脚本目录，服务于工程维护与运营效率；不承载业务逻辑。

## 包含内容
- 当前为空目录，预留运维、迁移、数据核查等辅助脚本。

## 与其它目录的依赖关系
- 允许依赖：`/packages/shared` 或 `packages` 中的通用接口（如需）。
- 禁止依赖：`/apps/*` 的 UI 或控制器逻辑，避免耦合业务流程。

## 禁止事项
- 未经审查的高风险脚本进入主分支。
- 缺少风险评估或回滚说明的脚本。

## 常见变更场景
- 新增维护脚本：明确用途、影响范围与回滚策略。
- 迁移辅助：标注目标环境与前置条件。
- 运行方式更新：记录权限要求与安全注意事项。

## 命名规范/约定
- 脚本命名体现用途（如 `audit-export`、`db-backup`）。
- 同类脚本保持一致前缀，便于检索与审计。

## 测试/验证原则
- 先在隔离环境验证，再应用于生产。
- 变更需确保可回滚、可审计、可重复执行。
